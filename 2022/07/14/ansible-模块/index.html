lvol

LVOL (/usr/lib/python2.7/site-packages/ansible/modules/system/lvol.py)

This module creates, removes or resizes logical volumes.

  • This module is maintained by The Ansible Community
    OPTIONS (= is mandatory):

  • active
    Whether the volume is activate and visible to the host.
    [Default: yes]
    type: bool
    version_added: 2.2

  • force
    Shrink or remove operations of volumes requires this switch. Ensures that that filesystems get never corrupted/destroyed by
    mistake.
    [Default: no]
    type: bool
    version_added: 1.5

  • lv
    The name of the logical volume.
    [Default: (null)]

  • opts
    Free-form options to be passed to the lvcreate command.
    [Default: (null)]
    version_added: 2.0

  • pvs
    Comma separated list of physical volumes (e.g. /dev/sda,/dev/sdb).
    [Default: (null)]
    version_added: 2.2

  • resizefs
    Resize the underlying filesystem together with the logical volume.
    [Default: no]
    type: bool
    version_added: 2.5

  • shrink
    Shrink if current size is higher than size requested.
    [Default: yes]
    type: bool
    version_added: 2.2

  • size
    The size of the logical volume, according to lvcreate(8) –size, by default in megabytes or optionally with one of
    [bBsSkKmMgGtTpPeE] units; or according to lvcreate(8) –extents as a percentage of [VG|PVS|FREE]; Float values must begin
    with a digit. Resizing using percentage values was not supported prior to 2.1.
    [Default: (null)]

  • snapshot
    The name of the snapshot volume
    [Default: (null)]
    version_added: 2.1

  • state
    Control if the logical volume exists. If present' and the volume does not already exist then the size’ option is required.
    (Choices: absent, present)[Default: present]

  • thinpool
    The thin pool volume name. When you want to create a thin provisioned volume, specify a thin pool volume name.
    [Default: (null)]
    version_added: 2.5

  • vg
    The volume group this logical volume is part of.
    [Default: (null)]

NOTES:
* You must specify lv (when managing the state of logical volumes) or thinpool (when managing a thin provisioned
volume).
