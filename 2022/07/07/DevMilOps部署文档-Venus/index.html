将镜像上传至venus：
方法1：

方法2：
使用scp命令将包含所需镜像的压缩包传输至venus服务器：

  1. scp [压缩包文件路径] root@[服务器ip]:* *[路径]
  2. 解压缩：tar –(z)xvf [压缩包]
  3. docker push [所需镜像] 到venus镜像仓库

其他可能需要的docker命令:

  • 登录：docker login http://xxxxx.com
  • 登录私有hub创建项目，例如项目叫：abc-dev
  • 给镜像打tag：docker tag [已有镜像id] [新镜像名:tag]
  • 推送镜像到仓库：docker push [镜像名称] 注：镜像名应包含正确的镜像仓库地址
  • 将镜像保存到文件：docker save -o(输出到文件): docker save -o [文件名].tar 镜像名:tag*

创建持久化存储卷

\1. 连接venus服务器，选择节点：ssh [masterX、nodeX…\

\2. 创建存储卷目录：mkdir -p /data/[存储卷目录:如devops-jenkins] （目录名应注意区分命名空间）

  1. 进入venus界面：存储管理-存储类，若没有”local”类型的存储类，则新增local存储类：


若已存在存储类，则创建持久化存储卷：

进入venus应用市场，部署jenkins


替换jenkins插件：

  1. 在venus中将jenkins应用的副本设为0
  2. 将plugins.tar上传到jenkins持久化存储卷目录：/data/devops-jenkins；
    或将本地plugins.tar传输到持久化存储卷所在节点：scp plugins.tar root@[节点名称]:/data/devops-jenkins
  3. 连接venus服务器，ssh [节点名称] 切换到持久化存储卷目录：
    cd /data/devops-jenkins
    mv plugins plugins_backup
  4. 解压plugins.tar包：tar -xvf plugins.tar
  5. 在veunus中将jenkins应用的副本设为1

部署gitlab

创建gitlab持久化存储卷

\1. 连接venus服务器，选择节点：ssh [masterX、nodeX…]

\2. 创建存储卷目录：mkdir -p /data/[存储卷目录:如devops-gitlab]（目录名应注意区分命名空间）

  1. 进入venus界面创建持久化存储卷

进入venus应用市场，部署gitlab-ce


等待gitlab完全启动…

  1. 进入venus配置管理，编辑gitlab配置文件，删除alertmanager、letsencrypt项

  1. 将devops命名空间的gitlab应用副本设为0

mkdir devops-gitlab

  1. 重新发布gitlab应用，选择9.5.1的tag号，将应用副本重新修改为1
