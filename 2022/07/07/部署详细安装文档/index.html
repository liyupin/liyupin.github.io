一键部署详细安装文档

一键部署详细安装文档

本文档主要用于帮助用户在虚拟机或物理机上安装的核心组件与安装web客户端。

概览

术语解释

配置要求

  • 硬件需求

    主机：

    | 组件 | 最小CPU | 最小内存 | 作用 | | ————– | ——- | ——– | —————————- | | 数据节点 data | 4 Core | 8 Gi | 主要用于存储系统数据 | | coreserver节点 coreserver节点 | 1 Core | 1 Gi | 主要用于运行coreserver节点程序 | | bootstrap节点 bootstrap节点 | 1 Core | 1 Gi | 主要用于运行bootstrap节点程序 | | hub节点 hub | 1 Core | 1 Gi | 主要用于运行hub程序 | | web节点 web | 1 Core | 1 Gi | 主要用于运行web程序 | | log节点 log | 1 Core | 1 Gi | 主要用于运行log程序 | | etcd节点 etcd | 1 Core | 1 Gi | 主要用于运行etcd程序 | | proxy节点 proxy | 1 Core | 1 Gi | 主要用于运行proxy程序 | | seam节点 seam | 1 Core | 1 Gi | 主要用于运行seam程序 | | agent节点 agent | 1 Core | 1 Gi | 主要用于运行agent程序 |

    注：

    • 至少需要两台Linux主机，其中一台作为数据存储与镜像仓库共用节点，一台作为docker节点。

  • 软件需求

    | 软件 | 版本 | | —— | —————————– | | docker | 不跨大版本的情况下，>=19.03.8 | | etcd | 不跨大版本的情况下，>=3.3.10 | | rqlite | 不跨大版本的情况下，>=5.1.0 | | nginx | 不跨大版本的情况下，>=1.16.1 |

  • 系统兼容性

    | 架构 | 系统 | | —– | ———- | | amd64 + arm64 | Ubuntu 18.04 LTS |

注意事项

  • 所有主机应当为全新环境。
  • 本文档中所有命令示例，都是在安装包中install目录下执行的。
  • 所有bash命令中，不同开头含义如下：
    • $：其后是需要执行的命令
    • #：其后是注释
    • 无：其后是上一命令输出或交互式命令的内容
  • 数据节点与日志节点只能有一个
  • 所有命令需以root权限运行
  • 集群内所有节点，除文档中的各程序外，尽量不以任何方式，启动其他程序
  • 可以将各主机DNS换成对于自身较快的DNS服务器，加快下载速度
  • 由于网络原因，脚本执行可能出现错误，此时重新执行该脚本即可
  • 由于pdf格式限制，请不要直接复制代码执行，若想直接复制，可以查看md格式文档
  • 由于需要为局域网内提供自动加入集群功能，需确保UDP协议的8873端口未被占用

核心组件安装

当前系统安装包分为在线与离线两个版本。若想使用在线安装包（即与此文档同级目录中的online.tgz文件），则必须可以连接到公司的镜像仓库（registry.dockercloud.cn ），否则，请使用离线安装包（即与此文档同级目录中的offline.tgz文件）。

架构 在线安装包 离线安装包 离线安装包所需系统资源 私有仓库基础镜像

根据需要下载完安装包后，解压安装包文件（online.tgz或offline.tgz文件），得到一个名为install目录，通过执行此目录下安装程序，即可完成安装。

注意：

  • 若使用离线安装包，请同时下载此文档同级目录中的ben.tar，并将其拷贝到解压后的install目录下

编辑配置文件

编辑install目录下的config.yaml文件.

日志等级

编辑config.yaml中的logLevel字段,默认 info,设置此字段后，ben、web、hub、组件中的日志等级将会被设置成相应的日志等级。

启动节点ip

编辑config.yaml中的localIp,设置为启动节点的ip,此ip需要集群中的所有节点均可访问.

集群中的镜像仓库

如果有必要，请修改集群中启动的registry domain.

镜像设置

如果使用公司内网仓库,或者仓库中有全量的安装镜像,请设置existRegistryInfo.

如果是使用的镜像仓库中的镜像,请更改为对应的镜像仓库名称,如图所示是使用镜像仓库中的镜像安装.

若使用完全离线安装的方式，请将各镜像中的地址，替换为设置的registryDomain。

节点设置

需要在配置文件中填入节点的信息,用于安装程序使用.除了基本的节点账号信息,还需要填入节点的组件信息,目前集群由一下组件组成:

组件 组件说明
data 安装程序会在这个节点上启动集群的数据服务,包含etcd,rqlite和镜像仓库,节点要求见配置要求
coreserver 这是集群的核心组件,提供资源操作api,节点要求见配置要求
bootstrap 这是集群的核心组件,提供资源控制,节点要求见配置要求
web 提供页面访问的组件,节点要求见配置要求
proxy 这是对外暴露集群服务的组件,装有此组件的节点可对外暴露服务,节点要求见配置要求
seam 这是docker跨节点网络插件,节点要求见配置要求
log 这是集群中的日志收集组件,所有组件产生的日志会由此组件收集,节点要求见配置要求
etcd 这是集群数据库节点,节点要求见配置要求

所以data节点的配置尽量可以高一点达到4C8G(cpu4 核,内存8G),其他组件的要求在1C1G即可.
